# 从表征学习角度看 Being-H0.7、Fast-WAM 与 π0.7：VLA和WAM标签之下，都在做类似的事

---

近期 VLA 和 WAM 的路线之争一直被广泛关注，然而，正如 GeneralistAI 的博客所强调的，这些"流行词"可能容易把人带歪，它们将方法简化为标签，总是让人误认为这是一个双选题或者三选题。VLA 和 WAM 固然在模型结构上有所差别，但我们应该关注一些更为本质的事，从表征学习的角度重新审视这些方法。

有幸和清华大学世界模型水木学者、智源研究院世界模型智源学者崔森老师交流了这一想法，崔森老师的想法非常 impressive，从表征学习的视角，我们重新审视了近期的几篇热门工作：**Being-H0.7、Fast-WAM 与 π0.7**。

这些论文表面上看路线差异很大：π0.7 仍然是 VLA 路线，强调 diverse context conditioning；Fast-WAM 站在 WAM 这边，讨论推理时是否还需要生成视频；Being-H0.7 则提出 Latent WAM，用 future-aware posterior branch 去塑造 latent reasoning space。

但如果从表征学习的角度看，这三篇工作其实在关注非常相似的事：**模型到底应该基于什么样的中间表征做决策？什么样的特征，才足以支撑机器人生成好的动作？**

---

## 什么样的表征才是"好表征"

同一个视觉场景下，"拿起杯子"可能对应很多可行动作：从左边绕过去、从右边绕过去、先调整手腕、先推开障碍物等等。语言指令通常只描述目标，不描述路径；动作轨迹虽然提供了路径，但它只是一条具体实现，而不是告诉模型"为什么这个动作在这个场景下是合理的"。（PI 的博客 Moravec's Paradox 中也提到，人们不会在语言中详细描述一个动作应该怎么做）

这会带来一个问题：如果模型只看到当前观测、语言和动作，它很容易学到表面的相关性，而不是学到真正支撑动作的物理、时序和策略结构，导致行为坍塌——有限且稀疏的动作监督会阻止模型学习真正有物理意义且多样的表征。π0.7 也从另一个角度指出，同一个任务在不同质量、速度、策略、embodiment 下对应不同数据，如果不提供足够上下文，模型可能把混合质量数据平均掉，生成次优行为。

以上的问题说明了如果只使用当前观测、语言和动作数据，无法得到一个足够好的表征。许多 WAM 的方法带来了一个启发——使用未来观测来训练中间表征。然而，这种方式也存在一个问题——完整地预测视频难度本身太大了，对于动作生成来说，预测完整的视频是没必要的。π0.7 等工作带来的一个启发是——**预测关键帧或许足够得到良好的中间表征**。

接下来我们可以看看这三篇工作分别是怎么做的：

---

## Being-H0.7

Being-H0.7 的方法最直接地把这个问题写成了模型内部的 latent 表征问题。它认为像素级 WAM 的问题在于，未来演化当然重要，但在像素空间里显式建模未来往往昂贵、噪声大，而且和下游控制只是弱对齐。机器人不一定需要知道未来每个像素长什么样，它更需要知道哪些未来变化会影响动作。

这里 $I$ 是指令，$O_{1:H}$ 是观测历史，$S$ 是状态，$z$ 是 latent queries，$A$ 是动作 chunk。$z$ 的作用不是简单多加几个 token，而是在 Transformer 层间传播中形成一个显式 latent reasoning space：模型先把当前上下文整理成一个中间 latent state，再用它指导动作生成。

Being-H0.7 同时引入训练期 posterior branch：prior branch 只能看当前上下文和 latent queries，posterior branch 额外看到未来观测，并把 latent queries 替换成 future embeddings。然后通过对齐隐藏状态，让 prior latent 去靠近 posterior latent。

简单来说，posterior branch 是一个训练期老师，使得模型学会从未来观测中提取哪些对动作有用的信息；prior branch 则在推理时只看当前上下文，但通过训练学会了这种未来感知的表征。

---

## Fast-WAM

Fast-WAM 回答了一个问题：WAM 推理时是否真的需要显式预测未来视频，还是训练期已经把好表征学出来了？

论文的回答很明确：**主要价值可能来自 video co-training 对表征的塑造，而不是推理时显式生成未来观测。**

Fast-WAM 通过 attention mask 来实现这一点，未来的 video tokens 和 action tokens 互相都不能 attend 到，避免未来信息直接泄漏进动作分支。

视频预测和动作预测所共享的上下文（当前观测、语言）通过在训练时进行视频动作联合训练，得到了一个具有未来感知的特征 $z$，这也使得不必在推理时显式地生成视频，仅靠这个良好的表征就可以生成好的动作。

---

## π0.7

π0.7 看起来和前两者差异最大，从结构上它主体是 VLA 而非 WAM，在实现方式上，它更偏向于在数据层面构造所需的关键信息（包括关键未来帧以及用于利用数据多样性的 metadata），显式地放进 context。

传统 VLA 的 context 往往只是语言指令：$c = \{L\}$  
π0.7 则把 context 扩展为任务指令、子任务指令、subgoal images、episode metadata、control mode 等多种条件。它的训练目标仍然可以写成根据观测历史和上下文预测动作 chunk：

$$\pi_\theta(a_{t:t+H} | o_{1:t}, c)$$

π0.7 的几个 context 组件都可以从特征构造的角度理解：

- **子任务指令**把长程任务拆成当前阶段。它把"最终目标"转化成"当前应该做什么"的局部决策特征，减少长程任务中的语义歧义。
- **subgoal images** 把语言难以描述的空间状态变成视觉目标。它不要求动作模型自己从一句话里推断"干净折好的衣服应该长什么样"，而是把近未来目标状态直接作为输入特征提供。
- **episode metadata** 把轨迹质量、速度、是否犯错这些隐藏变量显式化。它告诉模型哪些数据是高质量策略，哪些数据只是失败或低质量 rollout。没有这些标签，混合质量数据可能互相污染；有了这些标签，低质量数据也能被正确地上下文化。
- **control mode** 则把动作空间本身放进 prompt，让模型知道当前要用 joint 格式还是 eef 格式。

π0.7 利用"未来观测"的方式和 Being-H0.7、Fast-WAM 差别比较大，它使用了一个单独的世界模型 BAGEL 用来生成 subgoal images（比起预测未来的每一帧或者连续的一段视频，subgoal images 这种方式更为精巧，动作转换的关键帧对于动作生成或许更有意义），这些 subgoal images 直接作为输入 prompt 的一部分，从数据入口侧入手，得到对未来关键帧有感知能力的良好表征。

## 总结

在笔者看来，接下来具身基础模型的竞争重点，不会停留在"VLA 还是 WAM"的争论上，而会落到更本质的问题：**怎样更好更稳定地构造表征**——谁能更好地定义这个中间表征，谁就更可能在真实机器人任务上获得稳定的泛化能力。

本文通过三篇经典工作，说明了什么是好的表征，应该利用什么样的数据来得到好的表征。这三篇工作，不管主体模型属于 WAM 还是 VLA，它们都在收敛到一个具体的共识上：**中间表征不仅需要动作来监督，还需要未来关键信息来监督**。目前主要利用到的关键信息在于未来的关键帧（不管在像素空间还是潜空间），而当下的一个急需解决的问题则在于如何更好地利用关键信息来构建我们需要的好的表征——这是与 VLM、WAM 无关的问题，也是真正核心的问题。
