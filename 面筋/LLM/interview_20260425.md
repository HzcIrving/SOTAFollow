## 日期：20260425

### LLM 领域

> 今日无新增特定论文，以下为通用领域思考题，聚焦于 FM 基础知识和 LLM 核心机制。

**通用领域问题：LLM 与 Foundation Model 基础**

1. **[基础概念题]** 请解释 RoPE（Rotary Position Embedding）相比绝对位置编码（AP）和相对位置编码（Tuau-Detrta）的核心改进，以及它如何支持长上下文建模。
   - **答案/思考提示**：RoPE 将位置信息编码为旋转矩阵，而不是像 AP 那样直接加到 token 嵌入上，或像 T5 那样的相对位置偏置。核心洞察：给 query 和 key 向量旋转角度 θ_i = i * θ_1，其中 θ_1 = 1/10000^{2/d}，则两个位置 m 和 n 的 attention score 对应旋转角度的差值：q_m · k_n 仅依赖于相对位置 (m-n)。这种设计的优势：(1) 位置信息通过连续旋转注入，不会因位置编码外推而失效；(2) 可以通过"线性插值"（如 YaRN）或"NTK-aware scaling"在推理时泛化到更长上下文；(3) 与 FlashAttention 等高效注意力实现兼容（无需修改 attention mask）。

2. **[深度思考题]** LLM 的 scaling law 表明模型性能和算力、数据量、参数量呈幂律关系。请从"表达能力"和"训练效率"两个角度解释：为什么增大模型规模比单纯增加数据更能提升模型能力？
   - **答案/思考提示**：
     - **表达能力角度**：Transformer 的表达能力随参数量呈指数增长（单层注意力可表达 O(N²) 的关系，多层堆叠可表达更深层的语义组合）。更大的模型拥有更多的 attention heads 和更宽的隐藏层，可以同时建模更多的独立特征和更长的依赖链。数据量翻倍只是增加了样本多样性，但模型容量（参数量）决定了模型能从数据中提取多少知识。
     - **训练效率角度**：大模型具有更高的 sample efficiency——相同数据在大模型上能学习到更多层次的表示，因为每一层都在对前一层的表示进行组合。这是"双重下降"现象背后的原因之一：小模型容易欠拟合，大模型更容易吸收数据的模式。但过大的模型（如训练数据量不足）会陷入"过度参数化"状态，表现为训练 loss 下降但泛化差。
     - **涌现能力（Emergent Abilities）**：某些复杂推理任务只有在模型规模超过临界阈值后才会出现，这暗示大模型能够以新的方式组合已有知识，而非简单线性外推。

3. **[深度思考题]** 在 LLM post-training 阶段，RLHF（Reinforcement Learning from Human Feedback）和 SFT（Supervised Fine-Tuning）的本质区别是什么？为什么很多 LLM（如 DeepSeek-R1）选择用 RLHF 而非更多 SFT 来激发推理能力？
   - **答案/思考提示**：SFT 是标准的监督学习——给定 prompt 和对应的"理想响应"，用交叉熵损失让模型学习拟合这个映射。RLHF 则引入 reward 信号，让模型学习"什么样的响应是好的"，而非"什么样的响应是正确的"。核心区别：SFT 给出的是**显式的、确定性的**参考答案；RLHF 给出的是**隐式的、分布级的**偏好信号。
     - 推理能力的涌现依赖于模型发现"正确的思维链"，而不仅仅是模仿示例。SFT 需要人工设计正确推理的示例，成本高且容易被错误示例带偏；RLHF 通过 reward 机制，可以探索更广阔的解空间，找到人工难以设计的推理路径。DeepSeek-R1 的 GRPO 算法让模型在 group 内比较不同解的质量，reward 信号驱动模型发现并强化"能导向正确答案的思维链"，这是 SFT 的"模仿"机制所无法实现的。

4. **[实践应用题]** 假设你需要对一个 7B 参数的 LLM 做 PEFT（Parameter-Efficient Fine-Tuning），有 LoRA、QLoRA、Prefix Tuning 三种选择。请从"显存占用"、"微调速度"、"下游任务效果"、"适用场景"四个维度进行比较，并给出选择建议。
   - **答案/思考提示**：
     - **LoRA**：在 Attention 的 Q/K/V/O 矩阵上附加低秩分解矩阵 A 和 B。显存占用：额外约 0.1%~1% 的原始模型参数量（对 7B 约几十 MB）。微调速度：与全参微调相近（需前向所有参数但只更新 LoRA 参数的梯度）。效果：通常可达到全参微调的 95%+。适用：通用下游任务、简单分类/生成。
     - **QLoRA**：LoRA + 4-bit NF4 量化 + 分页优化器。显存占用：7B 模型可加载到单卡 24GB（原本需要 14GB+ × 4 = 56GB）。微调速度：显著慢于 BF16 LoRA（反量化开销）。效果：与全量 BF16 微调相当。适用：没有多卡资源时的全参数量级任务。
     - **Prefix Tuning**：在每层 Transformer 输入处添加可学习的前缀 tokens。显存占用：约 0.1%~3%（取决于 prefix 长度）。微调速度：最快（前缀长度通常 10-100 tokens）。效果：对某些任务（尤其是需要"记住"前缀内容的）效果接近 LoRA，但通用性略差。适用：提示学习场景、知识注入。
     - **选择建议**：通用场景优先 LoRA；单卡 24GB 预算用 QLoRA；需要极快迭代且任务简单用 Prefix Tuning。

5. **[优缺点对比题]** 请对比自回归语言模型（如 GPT 系列）和 Prefix LM / encoder-decoder 架构（如 T5、BERT）在建模方式上的根本差异，以及这种差异如何影响它们在长上下文理解任务中的表现。
   - **答案/思考提示**：
     - **自回归（AR）**：每个位置只能 attend 到左侧上下文，输出是自左向右生成的。训练目标：最大化 P(x_t | x_{<t}) 的似然。优势：适合生成任务，loss 与推理分布一致；模型规模与生成质量直接挂钩。劣势：双向信息（如理解整个文档后的摘要生成）需要额外的设计（如思维链）。
     - **Encoder-Decoder（如 T5）**：输入序列通过双向 attention 建模，输出序列自回归生成。适合：需要完整理解输入后进行操作的任务（如翻译、摘要）。问题：encoder 存在信息 bottleneck——输入的全部信息需压缩到固定维度的隐藏状态，长输入时信息密度不足。
     - **Prefix LM（如 UniLM）**：输入部分双向 attention，生成部分单向 attention，本质是 AR 和 Encoder-Decoder 的折中。长上下文场景：Prefix LM 在输入阶段能利用双向 attention，但生成阶段与 AR 相同。对需要"先理解再生成"的任务（如检索增强生成），Prefix LM 通常优于纯 AR，因为它能更高效地压缩长输入信息。
     - **趋势**：近期 LLM 主流是纯 AR（GPT 架构），通过更长的上下文窗口和 Chain-of-Thought 来弥补双向理解的不足；encoder-decoder 架构在小规模模型和特定 NLU 任务中仍有优势。
