1. **[基础概念题]** PPO 的 Clipped Surrogate Objective 的核心思想是什么？为什么要用 min 操作？
   - **答案/思考提示**：`min(r_t(θ)Â_t, clip(r_t(θ), 1-ε, 1+ε)Â_t)` 中，当 r_t 在可行区间内时用原始目标鼓励策略提升，当 r_t 超出区间时用截断目标限制更新幅度。"min"操作确保即使 ratio 有利也不会被过度优化，从而实现策略的单调提升保证。关键洞察：clip 相当于在目标函数中引入了一个"悲观边界"。

2. **[深度思考题]** GRPO 用 group 内样本的均值和标准差估计 advantage，为什么这种相对比较比使用全局 value function 更适合 LLM 场景？
   - **答案/思考提示**：LLM 的 reward 通常是离散的验证结果（正确/错误），而非连续的光滑信号。Value Model 需要大量数据拟合复杂的 reward 地形，且在 sparse reward 场景下训练不稳定。Group-relative 估计本质上是同组内的排名比较——如果同题型的多解法中某解更优，则提升其采样概率。这种方法无需额外的模型容量，且天然对 reward scale 不敏感。但局限在于：若 group 内样本质量都很差（早期训练），相对比较无意义。

3. **[深度思考题]** NPO 提出用"近未来 checkpoint"作为辅助轨迹源，其核心洞察 S(Δ) = Q(Δ)/V(Δ) 的物理含义是什么？为什么 S 是凹函数（先升后降）？
   - **答案/思考提示**：Q(Δ) 随 Δ 增大而提升（更远的 checkpoint 越强），但 V(Δ) 呈指数增长（参数漂移导致 importance weighting 方差爆炸）。在训练早期，Q 很小但 V 也很小，S 处于低位；Δ 增大让 Q 快速提升而 V 增长尚缓，S 上升；当 Δ 过大，V 的指数增长主导，S 开始下降。因此存在唯一内点最优 Δ*。这本质上是一个"偏差-方差权衡"在时间维度上的体现。

4. **[实践应用题]** 假设你在训练一个代码生成模型，当前策略 pass@1 卡在 45% 不动，rollout 多样性枯竭。请设计一个基于 NPO 思想的自适应训练策略，需要包括：如何检测 plateau、如何选择 Δ、以及如何混合使用。
   - **答案/思考提示**：
     - **检测 plateau**：监控训练 reward 的 EMA 曲线 + policy entropy，当 reward 停滞且 entropy 下降时进入"警惕阶段"，再做一次 probe rollout 确认。
     - **选择 Δ**：在候选 Δ 集合 {5, 10, 20, 50, 100} 中，对每个 Δ 计算经验 Ŝ(Δ) = Q̂(Δ)/V̂(Δ)，选择使 Ŝ 最大化的 Δ*（Q̂ 用 probe rollout 在 mistake pool 上的 pass-rate，V̂ 用 per-token KL 散度的指数估计）。
     - **混合使用**：仅对当前策略 fail 但 Δ* checkpoint 能成功的 prompts，将第 n 个 slot 替换为 guidance 轨迹；其余 prompts 保持纯 on-policy GRPO。比例由 gate threshold τ 控制。

5. **[优缺点对比题]** NPO 与 RLEP（远未来回放）都使用更早的 checkpoint 轨迹，但效果差异巨大。请从 S(Δ) 曲线的角度解释两者效果不同的根本原因。
   - **答案/思考提示**：RLEP 的 Δ 通常极大（完全训练好的模型），此时 Q(Δ) ≈ 1（信号质量接近完美），但 V(Δ) 爆炸（off-policy 极重，importance weighting 趋于 0 导致梯度方差无穷大）。S(Δ→∞) → 0，有效信号被方差淹没。NPO 则在 Δ* ≈ 20-70 步的近未来位置找到最优平衡点，Q 足够高（显著优于当前策略）的同时 V 保持在可控范围。核心洞见：**不是越强的 teacher 越好，而是最强且分布差距可接受的 teacher 最好**。

**论文：《Near-Future Policy Optimization (NPO)》**

6. **[基础概念题]** NPO 的核心机制是在 rollout group 中替换一个 slot 为近未来 checkpoint 的正确轨迹。请写出完整的 NPO 目标函数，并解释 on-policy slots 和 guidance slot 的概率比 ρ 是如何分别计算的。
   - **答案/思考提示**：对第 i 个 slot，若 i < n（on-policy），则 q_i = π(t)，ρ^q_{i,t}(θ) = πθ/π(t)；若 i = n（guidance slot），则 q_i = π(t+Δ)，ρ^q_{i,t}(θ) = πθ/π(t+Δ)。由于 π(t+Δ) 与 π(t) 足够接近，IS 修正系数接近 1，重要性采样的 bias 可以忽略不计。这正是 NPO 相比远未来回放的关键优势。

7. **[深度思考题]** NPO 论文指出"𝒮 随 Δ 呈凹型（先升后降）"，但这个结论高度依赖于具体的训练阶段和模型状态。如果在训练初期（策略还很弱）使用 NPO，可能会出现什么特殊情况？
   - **答案/思考提示**：训练初期 Q(Δ) 曲线整体很低——即使近未来 checkpoint 也几乎无法正确解决任何问题。但更关键的问题是，此时 π(t+Δ) 与 π(t) 的分布差距实际上反而可能更小（因为两个 checkpoint 都很弱，能力都接近随机初始化），所以 V(Δ) 可能也不大。S 曲线的形状会随训练动态变化，这解释了为什么 AutoNPO 需要持续监控和重新搜索 Δ*。论文的 Early-Stage Bootstrapping 就是针对这个问题的解法：先用短 scout run 建立一个稍强的起点。
