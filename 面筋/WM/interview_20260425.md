## 日期：20260425

### WM 领域

**论文：《Being-H0.7: A Latent World-Action Model from Egocentric Videos》**

1. **[基础概念题]** Being-H0.7 的"潜在推理空间"（Latent Reasoning Space）是什么？它与直接用视觉特征作为动作预测条件的做法有何本质区别？
   - **答案/思考提示**：潜在推理空间是一组 K=16 个可学习的 tokens，插入在当前上下文 tokens 和动作 tokens 之间。与视觉特征的区别：视觉特征受预训练目标（图像重建/分类）主导，是"外观导向"的表示；而 latent queries 是"动作导向"的——它们仅受动作损失和对齐损失监督，无需对视觉重建负责。这保证了 latent 空间专门服务于动作决策，而非混杂视觉语义。类比：就像人类的"心理意象"不等同于看到的像素，而是对任务有意义的高层表征。

2. **[深度思考题]** Being-H0.7 的双分支设计（Prior-Posterior Branch）中，对齐损失 L_align 实际上在扮演什么角色？为什么说它本质上是一种"隐式蒸馏"？
   - **答案/思考提示**：后验分支拥有未来观测信息，先验分支没有。对齐损失强迫 h_prior ≈ h_posterior，使得先验分支在训练过程中逐步学会"在当前上下文的条件下，生成与拥有未来信息时等效的潜在表示"。这与蒸馏相同点：都是用更强教师信号指导学生；不同点：蒸馏是显式输出匹配，这里是对齐潜在空间，且学生（prior）和教师（posterior）共享同一 backbone 参数。推理时只用 prior，相当于"蒸馏后的学生网络"，无需教师。

3. **[深度思考题]** 论文在 Section 3.1 明确指出"不能用 visual-derived latent 替代 learnable latent"，理由是"弱表示坍缩"和"视觉语义主导"。请从优化目标的角度深入解释这个问题。
   - **答案/思考提示**：若 latent 从 ViT 视觉 token 派生，它同时受到两个目标的梯度影响：(1) 视觉编码器的原始预训练损失（重建/分类），(2) 动作头的 flow matching 损失。两个目标的梯度方向不一致，会产生干扰——视觉 token 需要保留细粒度纹理信息，而动作预测需要的是高层任务相关语义。learnable latent 只受动作相关损失驱动，梯度路径纯净。另外，若 latent 来自视觉编码器，则先验和后验分支天然共享视觉先验，难以解耦"未来外观预测"和"未来动作决策"——这是 JALA 等方法的局限。

4. **[实践应用题]** Being-H0.7 使用 MoT（Mixture of Transformers）打包策略，将双分支打包到单次前向传播中。请设计注意力掩码矩阵，并解释为什么这种设计能在单次前向中区分先验和后验 tokens。
   - **答案/思考提示**：掩码原则：共享上下文（指令/观测/状态）对所有 tokens 可见；先验 tokens（latent queries + 动作）与后验 tokens（未来嵌入 + 动作）相互隔离。矩阵：行为 query，列为 key/value。共享上下文行全 ✓；Prior-Specific 行对 Prior-Specific ✓ 对 Posterior-Specific ✗；Posterior-Specific 行对 Posterior-Specific ✓ 对 Prior-Specific ✗。关键：两分支使用相同的 token 位置 ID（位置编码共享），所以对齐损失作用在对应位置 hidden states 上时，结构上完全对齐。计算效率提升约 2x（一次 backbone 前向替代两次）。

5. **[优缺点对比题]** Being-H0.7 与 Fast-WAM 都声称"推理时不需要像素级 rollout"，但实现机制完全不同。请从"世界模型的形式"和"训练目标"两个维度对比两者的核心差异。
   - **答案/思考提示**：
     - **世界模型形式**：Fast-WAM 的世界模型是"视频联合去噪"——训练时视频和动作一起重建，模型学到物理直觉；推理时直接跳过想象步骤，用观测单次前向输出动作（"训练即一切"）。Being-H0.7 的世界模型是"潜在嵌入对齐"——训练时通过双分支对齐让 prior 分支隐式获得未来感知能力；推理时用 prior 分支，latent queries 作为未来信息的代理。
     - **训练目标**：Fast-WAM 是单目标（视频+动作联合重建），Being-H0.7 是多目标（flow matching + 对齐损失 + 正则化）。Fast-WAM 的效率来自"不需要世界模型推理时 rollout"，Being-H0.7 的效率来自"不需要像素级推理"。

**论文：《Epona: Autoregressive Diffusion World Model for Autonomous Driving》**

6. **[基础概念题]** Epona 的核心创新是"解耦时空因子分解"——GPT-style Transformer 处理时间动态，DiT 处理空间渲染。请解释为什么这种解耦是必要的，以及如果不解耦会有什么后果？
   - **答案/思考提示**：视频扩散模型的联合分布建模面临"长期一致性"和"视觉质量"的冲突：全局时序建模需要关注长程依赖，但每帧空间渲染也需要高分辨率细节。两者在特征层面需求不同：时序建模需要压缩的抽象表征（低分辨率高语义），空间渲染需要像素级细节。若不解耦，单一模型需同时满足两种需求，要么牺牲时序一致性（各自独立生成），要么牺牲视觉质量（压缩过度）。Epona 的解耦让时间预测在 64 个 latent tokens 上进行（高效且一致），空间渲染在每帧独立进行（高保真）。

7. **[深度思考题]** Chain-of-Forward 训练策略的本质是解决 Exposure Bias。请从梯度反向传播的角度分析：为什么传统的 Teacher Forcing 无法教会模型"纠错"，而 Chain-of-Forward 可以？
   - **答案/思考提示**：Teacher Forcing 训练时，每一步的输入总是真实历史，梯度只流经"正确路径"。模型从未见过自己错误的状态，所以推理时一旦有误差，误差会累积（因为模型没学过如何在错误状态基础上恢复）。Chain-of-Forward 强制模型在训练时就用自己上一步的预测作为下一步的输入，形成"错误状态→纠错"的梯度路径。多步 MSE loss 反向传播时，梯度会流经"将错误拉回正确轨道"的参数——模型学到了纠错函数。这与 DAgger 算法的思想一致：模仿学习必须在对抗性数据分布上训练才能泛化到推理分布。
